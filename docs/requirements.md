# 健身记录应用需求调研报告

## 1. 项目概述

### 1.1 项目背景

随着健康意识的提升，跨平台健身记录应用成为越来越多用户的刚需。本应用定位为一款支持手机（iOS/Android）和电脑（Web/桌面端）的健身记录工具，采用本地数据存储确保用户隐私，采用新拟态（Neumorphism）设计风格提供独特的视觉体验。

### 1.2 目标用户

- 健身爱好者（跑步、骑行、力量训练）
- 注重健康管理的普通用户
- 需要追踪体脂和基础代谢的用户

## 2. 行业产品功能调研

### 2.1 主流竞品分析

| 应用名称 | 核心功能 | 数据存储 | 特色 |
|---------|---------|---------|------|
| Keep | 课程+记录 | 云端 | 社区属性强 |
|悦跑圈 | 跑步记录 | 云端 | 跑步垂直领域 |
|薄荷健康 | 饮食+体脂 | 云端 | 食物库丰富 |
|Fitbit | 多运动+设备 | 云端+本地 | 硬件生态 |
|Apple Fitness+ | 运动记录 | iCloud | 苹果生态 |

### 2.2 行业功能趋势

1. **多运动类型支持**：跑步、骑行、游泳、健身、瑜伽等
2. **智能数据统计**：趋势图表、周期筛选、个人记录
3. **身体指标追踪**：体脂率、BMI、基础代谢、肌肉量
4. **离线优先**：本地存储优先，云端同步为辅
5. **个性化主题**：深色/浅色/自定义主题

## 4. 技术选型建议

### 4.1 框架选择

| 平台 | 推荐技术 | 理由 |
|-----|---------|------|
| 移动端 | React Native / Flutter | 跨平台原生体验 |
| 桌面端/Web | Tauri + React | 轻量级、性能好 |

### 4.2 数据存储

- **移动端**：SQLite + WatermelonDB
- **桌面端/Web**：IndexedDB + electron-store

### 4.3 设计风格

新拟态（Neumorphism）核心特点：
- 柔和的凸起/凹陷阴影效果
- 低对比度、单色调配色
- 触摸感知的立体效果

## 6. 附录：核心计算公式

### 6.1 Mifflin-St Jeor 基础代谢公式

```
男性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5
女性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161
```

### 6.2 体脂率估算公式（基于BMI）

```
男性体脂率 = (1.20 × BMI) + (0.23 × 年龄) - 16.2
女性体脂率 = (1.20 × BMI) + (0.23 × 年龄) - 5.4
```

### 6.3 BMI计算公式

```
BMI = 体重(kg) / 身高(m)²
```
